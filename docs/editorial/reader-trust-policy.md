# Reader Trust Policy

KOTATSUでは、公開画面に出る言葉と見え方が、読者に誤解や不快感を与えないことを重視する。この文書は読者向け表示の正本であり、画像採否の詳細は `docs/editorial/ai-visual-policy.md` に委ねる。

## Principles

- 未完成の記事を完成済みのように見せず、公開前の記事ページへリンクしない。
- 制作内部の役割、GitHub、prompt、metadataを読者向け本文へ出さない。
- AI生成であることは隠さないが、制作工程を主役にしない。
- 読者を急がせる、煽る、正解を押しつける表現を避ける。
- 架空の人物、場所、道具、ビジュアルを、実在の取材・撮影・商品写真だと誤解させない。
- 季節外れの見え方、不衛生に見える私物配置、実在人物への類似など、読者信頼を損なう画像を公開しない。

## Reader-Facing Copy

次の語は制作管理には使えても、読者向け画面へそのまま出さない。

- 編集長、進行編集、Vol.設計、候補案
- GitHub Issue planning、AI生成ビジュアル方針
- NOW PLANNING、Coming Soon、Comming Soon
- Lorem ipsum、TODO
- 絶対買うべき、今すぐ真似したい、モテる、垢抜け、神アイテム
- 正解コーデ、これ一択、爆売れ、知らないと損、買わせる

未完成状態では「準備中」「公開準備中」「公開予定日」を優先する。「公開日が決まり次第お知らせします」と表示しても、記事ページへのリンクは作らない。

## Publication States

- `draft`: サイトに表示しない。
- `scheduled`: 目次やカードに公開予定として表示できるが、記事ページへリンクしない。
- `published`: 記事ページを生成し、リンクしてよい。

## AI Visual Disclosure

AI生成であることはalt、metadata、ページ共通のビジュアルクレジットで明示する。

sidecarの `caption` は編集記録であり、自動的に画面へ表示しない。読者向けfigcaptionを使う場合は、prompt、生成手順、内部用語を含めず、画像から読み取れる自然な記事文として別途編集する。

架空専属AIモデルは顔や全身を表示できるが、実在の取材対象、寄稿者、愛用品の所有者、発言者として見せない。

## Automated Check

`pnpm content:check` は、公開画面の内部用語、TODO、仮表示、煽り表現、未公開導線を検査する。画像そのものの季節感、類似、人物、衛生的な配置はビジュアル編集と進行編集が実画像で確認する。
