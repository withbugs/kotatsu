# Reader Trust Policy

KOTATSUでは、公開画面に出る言葉と見え方が、読者に誤解や不快感を与えないことを重視する。

## 基本方針

- 未完成の記事を完成済みのように見せない。
- 公開前の記事ページへリンクしない。
- 制作内部の都合や役割名を、読者向けの本文や画面文言に出しすぎない。
- AI生成ビジュアルは隠さないが、制作工程そのものを前面に出しすぎない。
- 読者を急がせる、煽る、正解を押しつける表現を避ける。
- 架空の人物、場所、道具、ビジュアルを、実在の取材・撮影・商品写真だと誤解させない。
- 発行時期と明らかに異なる服装、素材、光、天候、小物を使い、読者に季節を誤認させない。

## 公開画面で避ける表現

- 編集長
- 進行編集
- Vol.設計
- AI生成ビジュアル方針
- 候補案
- GitHub Issue planning
- NOW PLANNING
- Coming Soon / Comming Soon
- Lorem ipsum
- TODO
- 絶対買うべき
- 今すぐ真似したい
- モテる
- 垢抜け
- 神アイテム
- 正解コーデ
- これ一択
- 爆売れ
- 知らないと損
- 買わせる

これらは制作管理上の言葉としては使ってよいが、読者が見る画面では、次のような言葉に置き換える。

- 準備中
- 公開準備中
- 公開日が決まり次第お知らせします
- 記事ラインナップは準備中です
- 公開前の記事ページにはリンクしません

## 未完成記事の扱い

- `draft`: サイトに表示しない。
- `scheduled`: 目次やカードには表示してよいが、記事ページへリンクしない。
- `published`: 記事ページを生成し、リンクしてよい。

## AI生成ビジュアルの扱い

AI生成であることは、alt、メタデータ、ビジュアルクレジットで明示する。ただし、トップページや未完成状態の案内文では、制作工程の説明を長く出さず、読者には「準備中」「公開予定」の情報を優先して伝える。

AI生成画像は、プロンプトやメタデータの説明ではなく、生成後の実画像を基準に季節感と直近画像との差を確認する。発行月に対して不自然な重ね着、素材、色温度、天候が見える場合や、直近のheroと構図や色調が過度に似る場合は、読者向けに公開する前に再生成または差し戻す。

登録済みの架空専属AIモデルは、顔や全身を含めて使用してよい。ただし実在の取材対象、寄稿者、愛用品の所有者、発言者として見せない。実在人物や著名人の参照は禁止し、似て見える結果は採用しない。記事のaltまたはビジュアルクレジットではAI生成物であることを明示する。

## チェック

`pnpm content:check` は、公開画面に内部用語、未処理のTODO、仮表示、煽り表現が混入していないかを検査する。対象は `src/pages`, `src/components`, `src/layouts`, および `draft` ではないcontentである。
