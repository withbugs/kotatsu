# Approved GitHub Volume Plans

Approved volume plans are created only at the `planning:finalize` stage after the editor-in-chief reviews the latest research, locks the monthly theme and lineup, and records editorial approval.

They are the source of truth for article GitHub Issues, visual GitHub Issues, copy-editing GitHub Issues, and publishing GitHub Issues.

For Vol. 002 and later, an approved plan requires the corresponding candidate memo and must include:

- `編集承認日: YYYY-MM-DD`
- `## 調査サマリー`
- `## ウェブ需要シグナル`
- `## 読者需要の判断`
- `## テーマ`
- `## 季節感`
- `## 記事構成`
- `## AI生成ビジュアル方針`
- `## 執筆直前の再確認`

The plan must distinguish observed web demand from editorial judgment. Trends inform the decision, but the editor-in-chief must explain why the resulting theme fits KOTATSU and remains useful after the immediate trend passes.

Before each not-yet-started article enters its publication week, the editor-in-chief may refresh its brief using newer web evidence. The approved monthly theme is not reopened unless new information creates a factual, seasonal, safety, or reader-trust problem.

Naming:

- `vol-001.md`
- `vol-002.md`

Do not create or merge an approved plan during `planning:research` or `planning:shortlist`.
